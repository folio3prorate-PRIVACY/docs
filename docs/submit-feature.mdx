# Submit a Feature Request

Help shape the future of Octav by submitting feature requests and voting on ideas from the community.

## How It Works

1. **Search Existing Requests** - Check if your idea has already been suggested
2. **Submit Your Idea** - Create a detailed feature request
3. **Community Votes** - Other users upvote requests they want
4. **Team Reviews** - We review popular requests regularly
5. **Development** - Top requests are prioritized for development
6. **Launch** - Features are announced when released

## Submit Your Feature

### Feature Request Guidelines

**Good feature requests include:**
- Clear, descriptive title
- Detailed description of the feature
- Use case - why you need this feature
- Example scenario or user story
- Mockups or examples (if applicable)
- Priority level (nice to have vs. must have)

**Example of a great feature request:**

> **Title:** Portfolio Performance Comparison
>
> **Description:** Add ability to compare performance of multiple portfolios side-by-side in a single chart view.
>
> **Use Case:** As a user managing both long-term and trading portfolios, I want to compare their performance to understand which strategy is working better.
>
> **Example Scenario:** I have a "HODL Portfolio" and a "Trading Portfolio". I'd like to see a chart showing both portfolio values over the past year to compare their returns.
>
> **Priority:** Medium - Would significantly improve my workflow

## Feature Request Tips

### Increase Your Chances

**Be Specific**
- Detailed descriptions get more attention
- Vague requests are harder to implement
- Include concrete examples

**Show Value**
- Explain the benefit
- Describe the problem it solves
- Quantify the impact if possible

**Engage the Community**
- Share your request
- Ask others to upvote
- Respond to questions from the team

**Provide Context**
- Share your workflow
- Explain current limitations
- Suggest how it fits into existing features

### What We Look For

- **User Value** - Does it solve a real problem?
- **Demand** - How many users want this?
- **Alignment** - Fits with product vision?
- **Feasibility** - Technically possible?
- **Resources** - Development time required?

## Feature Development Process

### Our Process

1. **Collection** (Ongoing)
   - Gather requests from all channels
   - Deduplicate and categorize

2. **Review** (Monthly)
   - Team reviews new requests
   - Evaluate feasibility
   - Add status labels

3. **Planning** (Quarterly)
   - Prioritize based on votes and strategy
   - Add to product roadmap
   - Assign to sprints

4. **Development** (Ongoing)
   - Build and test features
   - Beta testing with select users
   - Iterate based on feedback

5. **Launch** (As Ready)
   - Release to production
   - Announce in changelog
   - Update feature request status

### Timeline Expectations

- **Small Features:** 1-2 months
- **Medium Features:** 2-4 months
- **Large Features:** 4-6+ months

*Note: Timelines are estimates and may vary based on complexity and priorities.*

## Frequently Asked Questions

**Q: How long until my feature is implemented?**
A: It depends on complexity, votes, and our roadmap. Popular requests are prioritized, but we can't guarantee timelines.

**Q: Can I submit multiple feature requests?**
A: Yes! Submit as many ideas as you have. Each request should be separate.

**Q: Will I be notified of updates?**
A: Yes, we notify requesters when status changes (via Discord or email, depending on submission method).

**Q: Can I contribute to development?**
A: We're exploring open-source contributions. Join Discord to discuss opportunities.

**Q: Why was my request declined?**
A: Requests may be declined if they don't align with our vision, have technical limitations, or have low community demand. We'll explain the reasoning.
